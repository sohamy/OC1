# 멀리 떨어진 친구와 플레이 — 릴레이 / 원격 연결

## 먼저 알아둘 것
직접 연결(KCP)은 **같은 PC(localhost)** 나 **같은 공유기(LAN)** 에서는 바로 된다.
인터넷 너머 친구와 하려면 NAT/방화벽 때문에 둘 중 하나가 필요하다:

- **릴레이**: 공인 IP를 가진 중계 서버가 양쪽 트래픽을 전달. (서버가 어딘가 있어야 함 = 무료라도 ‘서비스 가입’ 필요)
- **VPN(메시)**: 두 사람을 같은 가상 LAN으로 묶어 직접 연결처럼. (서버 불필요, 설치만)

게임에는 **접속 화면(ConnectUI)** 이 이미 들어있다 — 방 만들기 / 참가 / **서버 시작(전용)**.
NetworkManager 오브젝트에 `ConnectUI` 를 붙이고 `NetworkManagerHUD` 는 제거하면 된다.

## 옵션 0. 내 서버에 전용 서버(데디케이티드) — **코드 기본 지원, 추천(서버 보유 시)**
GameSession 이 데디 구조로 리팩터됨: **서버는 플레이어 없이 진행을 권위적으로 시퀀싱·중계**,
두 사람은 **둘 다 순수 클라**로 접속(슬롯은 접속 순서로 P1/P2 자동 배정).

1. **서버 빌드**: Build Settings 에서 Server/Headless(또는 일반) 빌드를 만들어 **공인 IP 서버**에 올림.
   - `OCNetworkManager` 에 헤드리스 자동시작 내장 → `-batchmode -nographics -server` 로 실행하면 자동 `StartServer()`.
   - 또는 에디터/빌드에서 접속 화면의 **서버 시작 (전용·데디케이티드)** 버튼.
2. **서버 포트 개방**: KCP 기본 `7777/udp` 를 서버 방화벽/보안그룹에서 연다. (클라는 포트포워딩 불필요)
3. **클라**: 접속 화면 주소 칸에 **서버의 공인 IP** 입력 → 참가. 두 명이 접속·커스텀하면 시작.
- 장점: 외부 서비스 0, 서버 권위적, 너가 가진 서버 그대로 사용. 단점: 서버 1대 상시 운영 + 포트 개방.

> 참고: 이건 ‘중계 릴레이’가 아니라 ‘전용 게임 서버’다. 둘 다 “서버가 공인 IP로 떠 있다”는 점은 같고,
> 데디는 서버가 게임 진행의 권위를 가진다(우리 구조). 릴레이(아래 B/C)는 바이트만 전달한다.

## 옵션 A. Tailscale (제일 쉬움, 무설정·무료) — 추천 시작점
코드 변경 0. 친구랑 오늘 바로 가능.

1. 두 사람 모두 **Tailscale** 설치 후 같은 계정/팀으로 로그인 (https://tailscale.com).
2. 호스트가 게임에서 **방 만들기(Host)**.
3. 호스트의 Tailscale IP 확인 (예: `100.x.y.z`, Tailscale 앱에 표시).
4. 게스트는 접속 화면 주소 칸에 그 `100.x.y.z` 입력 → **참가**.
   - KCP 기본 포트 `7777` 그대로 사용.
- 장점: 서버 불필요, 포트포워딩 불필요, 안정적. (ZeroTier도 동일하게 가능)

## 옵션 B. Edgegap Relay (관리형 릴레이, 무료 티어) — “진짜 릴레이”
중계 서버를 직접 안 굴려도 됨. Mirror 공식 파트너. **데디 서버 불필요 — 둘 중 한 명이 호스트.**
**→ 배포용 추천. `ConnectUI` 가 방 코드 흐름을 이미 지원하도록 연동돼 있음(아래).**

쓰는 트랜스포트는 `EdgegapLobbyKcpTransport`(로비 서비스 = 방 코드 발급/참가 자동) — Mirror 에 이미 번들됨.

**일회성 셋업(한 번만):**
1. https://app.edgegap.com 가입(무료 티어) → 대시보드에서 **API 토큰** 발급.
2. 씬의 NetworkManager 오브젝트에서 기존 `KcpTransport` 자리에 **`EdgegapLobbyKcpTransport`** 컴포넌트를 추가하고,
   `NetworkManager.transport` 와 `Transport.active` 가 이걸 가리키게 한다(기존 KCP 컴포넌트는 제거하거나 비활성).
3. 그 컴포넌트 인스펙터의 **"Create Lobby Service"** 버튼으로 로비 서비스를 배포 → `lobbyUrl` 칸이 자동으로 채워짐
   (API 토큰 입력 요구함). 이 URL 이 빌드에 박혀 나간다.

**플레이(매번):**
4. 호스트: 접속 화면 **방 만들기** → 화면 상단에 **방 코드**가 뜨고 **클립보드에 자동 복사**됨 → 친구에게 전달.
5. 게스트: 주소(코드) 칸에 그 **방 코드 붙여넣기** → **참가**. 릴레이 배정(최대 1분) 후 자동 연결.
- 장점: 데디 서버 0대·포트포워딩 0·받는 사람 추가설치 0, 전 세계 릴레이. 단점: 일회성 Edgegap 가입.
- 코드 연동은 `ConnectUI` 가 활성 트랜스포트가 `EdgegapLobbyKcpTransport` 면 자동으로 방 코드 흐름으로 전환(IP 흐름과 공존).

## 옵션 C. Light Reflective Mirror (LRM, 자체 호스팅 릴레이, 오픈소스 무료)
중계 서버를 직접 띄움(공인 IP 필요 — 저가/무료 클라우드 VM 한 대).

1. https://github.com/Derek-R-S/Light-Reflective-Mirror 의 서버를 VM에서 실행(공인 IP, 포트 개방).
2. 게임에 `LightReflectiveMirrorTransport` 임포트 → NetworkManager Transport 로 지정 → 릴레이 서버 IP 설정.
3. 호스트가 방을 만들면 **방 코드(Server ID)** 가 나오고, 게스트는 그 코드로 참가.
   - 이 경우 접속 화면의 ‘릴레이 코드’ 칸이 곧 방 코드가 된다.
- 장점: 완전 무료·방 코드 방식. 단점: 중계 VM을 직접 운영.

## 코드 연동 메모
- `ConnectUI` 는 **활성 트랜스포트 종류를 보고 자동 분기**한다:
  - `EdgegapLobbyKcpTransport` → 방 코드 흐름(호스트=방 만들기 시 코드 발급/복사, 게스트=코드 입력).
  - 그 외(KCP 등) → 기존 IP 흐름(주소 칸 = IP, `IP:포트` 지원).
- LRM처럼 `networkAddress` 를 방 코드로 쓰는 트랜스포트: 주소 칸에 코드 입력 → 그대로 동작.
- 게임 로직(`GameSession`)은 데디/호스트 양쪽 모두에서 동일하게 동작(슬롯은 접속 순서로 P1/P2 배정) — 변경 불필요.

## 추천
- **배포용(친구 몇 명, 추가설치 X)** → **옵션 B(Edgegap Lobby Relay)**. 데디 서버 없이 빌드 하나로 호스트/게스트.
- **지금 당장 테스트** → 옵션 A(Tailscale, 무설정).
- **완전 무료·자체 운영** → 옵션 C(LRM).
