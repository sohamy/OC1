# 자캐 커플 추리 비주얼 노벨 — 기획서

## Context (왜 만드는가)
자캐 커뮤(오리지널 캐릭터 롤플레이 커뮤니티)에서 활동하는 **커플 자캐**를 주제로 한 게임을 만든다.
두 사람이 각자 자기 자캐(커플)를 맡아, 개발자가 짠 **밀실 추리 사건**을 함께 풀어가는
**온라인 2인 협력 비주얼 노벨**이다. 핵심 가치는 "둘이서 같이, 자기 자캐로 RP 하며 사건을 푸는" 경험.

개발자는 게임 서버 프로그래머(코딩 능숙, Unity 선호). 기간은 약 **2주**.

## 확정된 결정 사항
| 항목 | 결정 |
|---|---|
| 장르 | 비주얼 노벨 (추리) |
| 플레이 방식 | **온라인 2인 협력 RP** (각자 기기에서 접속) |
| 엔진 | **Unity**, 데스크톱(.exe) 빌드 |
| 네트워킹 | **Mirror** (무료) |
| 다이얼로그 시스템 | **직접 구현** (ScriptableObject/JSON 데이터 기반) |
| 커플 커스텀 | 이름 + 사진. 한 장만 넣어도 되고, **표정/포즈 여러 장**을 넣어 장면별 전환도 가능 |
| 조연(용의자 등) | **전부 개발자가 작성**, 커스텀 없음 |
| RP 대사 | 각자 **자유 입력**(타이핑) → 양쪽 화면에 동기화 |
| 미니게임 | **2개** — ① 도입 요리(협력) ② 본편 증거 수집(협력) |
| 사건 컨셉 | **클래식 밀실 살인** — 눈 갇힌 별장. **피폐(무겁고 비극적인) 톤** |
| AI/외부 API | **사용 안 함** |

## 시나리오 — "눈은 모든 걸 덮는다" (밀실 살인 · 피폐)
**설정:** 커플(플레이어 자캐 2명)이 외딴 설산 별장 모임에 초대된다. 밤사이 눈보라로 별장이 완전히
고립되고, 첫날 밤 한 사람이 잠긴 방에서 시신으로 발견된다. 외부 침입 흔적은 위장. 범인은 안에 있다.

**핵심 피폐 장치:** 수사가 진행될수록 증거가 한쪽 자캐의 숨겨진 과거를 가리키고, 그 자캐가 유력 용의자로
몰린다. 연인(파트너)은 그를 믿을지 흔들린다. → 두 플레이어에게 '의심·불신·믿음'의 무거운 RP 갈등.

**장르:** 폐쇄형 별장 추리 / 연쇄살인 / 밀실 / 복수극. 폭설로 고립된 별장에 9명(+플레이어 커플)이 모이고
밤사이 **3명이 차례로 살해**된다(각 시신 곁 마른 꽃). 사망: 윤세현→배세린→심태오. **생존 7명**.

**등장(NPC 8, 플레이어 2):** 윤세현(주인·1번 피해자), 류기훈(옛 동업자), 한도희(관리인·**진범**),
배세린(목격자·2번 피해자), 강우진(의사), 오민재(기자), 차서윤(플로리스트), 심태오(보안기사·3번 피해자).
죽은 자도 산 자도 모두 5년 전 공방 화재에 얽혀 비밀이 있어, 범인을 쉽게 좁힐 수 없다.

**레드 헤링 구조:** 류기훈(원한·서재 근처)·강우진(사망시각 조작 가능)·오민재(사건 과다인지)·
차서윤(마른 꽃 출처)·한도희(열쇠 보유)가 모두 범인처럼 보인다. 한도희는 "너무 수상해 오히려 아닌 척".
**결정타** = 아무도 말하지 않은 '한도희의 동생'을 본인이 먼저 발설.

**단서 12개:** 깨진 창문·발자국 없는 눈밭·마른 꽃·멈춘 시계·{P1} 장신구(함정)·찢긴 장부·
꽃 주문 기록(익명)·CCTV 백업·관리인 신발 자국·여벌 열쇠(봉인 뜯김)·배세린 메모·발설된 비밀.

**9씬 구성:** 1.도착(모두 수상) 2.만찬+요리 미니게임 3.첫 살인·서재 수색(증거 미니게임)
4.조사/1차 추리(심문+추리 퍼즐) 5.2번째 살인(배세린)+믿음 선택 6.5년 전 화재 공개
7.3번째 살인(심태오)·연쇄 확정 8.생존자들의 거짓말(+동생 발설) 9.최종 추리(퍼즐 4종)→범인 지목.
**엔딩 3종:** 구원(정답+신뢰) / 진실의 값(정답+불신) / 파국(오답).

## 기술 설계
- **Unity 2022.3 LTS + Mirror.** 호스트(서버+클라) + 클라이언트. 턴제형이라 동기화 부담 작음.
- `GameSession`(NetworkBehaviour)에 진행 상태(SyncVar), 행동은 `[Command]`→`[ClientRpc]` 브로드캐스트.
- **동기화 대상:** 씬/비트 인덱스, 선택, RP 텍스트, 수집 단서, 범인 지목, 미니게임 상태.
- 핵심 시스템: Dialogue(데이터 기반 러너), Network(GameSession), Characters(사진 업로드+이미지 동기화),
  RP(자유 입력), Minigames(요리·증거), Mystery(단서노트·심문·지목·엔딩), UI.
- 외부 자산: Mirror(무료), StandaloneFileBrowser(무료, 데스크톱 파일 선택).

### 이미지 동기화(핵심 난점)
업로드 이미지는 로컬 파일 → 상대 클라가 못 봄. 업로드 시 리사이즈(가로~512px)+압축 바이트를
Mirror 메시지로 전송, 상대 측에서 Texture2D 복원.

## 2주 마일스톤
- D1–2: 프로젝트+Mirror 셋업, 접속, 씬 진행 동기화, 데이터 기반 대사창.
- D3–4: 커플 커스텀(사진·이름·표정) + 이미지 동기화.
- D5–6: RP 자유 입력 + 표정 전환 + 선택지/협력 결정 동기화.
- D7–8: 미니게임① 요리. D9: 미니게임② 증거 수집 + 단서 노트.
- D10–11: 추리 로직(심문·대조·지목·엔딩). D12: 시나리오 본문 데이터 완성(병행).
- D13: UI 폴리시·끊김 처리. D14: 빌드·2인 플레이테스트.

## 검증 방법
두 인스턴스(호스트·클라)로 접속해 ① 진행 동기화 ② RP 표시 ③ 사진 상대에게 보임 ④ 표정 동기화
⑤ 미니게임 협력 ⑥ 단서·심문·지목·엔딩 분기 ⑦ 끊김 복구를 확인. 도착→엔딩 1회 완주.
